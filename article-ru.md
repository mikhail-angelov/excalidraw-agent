# AI Agent Under the Hood: как я написал своего React за 5 минут, а потом научил его рисовать

*О чём молчат туториалы по LangChain, и почему ваш агент — это просто автодополнение с последствиями*

---

Недавно я задался вопросом: а что, если взять тот самый минимальный React из моей [старой статьи на Хабре](https://habr.com/p/415857/) и засунуть в него AI-агента? Ну, чтобы он рисовал диаграммы? Просто because we can.

Спойлер: я написал AI-агента в 90 строк кода. Он работает. Он пугает. Иногда кажется, что он осознаёт себя. Но он всего лишь цикл, который вы не замечаете.

## Завязка: каждому агенту нужен холст

Любой AI-агент — это, по сути, очень навязчивый попугай, которому дали инструменты. Попугай говорит: "Хочу нарисовать квадрат". Если у него есть клюв — он рисует. Если нет — он просто говорит "я бы нарисовал, но..." и ничего не происходит.

Поэтому сначала я сделал холст — минимальный сервер на Express, который хранит элементы Excalidraw и отдаёт их через REST API. Ничего лишнего:

```javascript
// server/index.js — минимальный canvas server

const elements = new Map()

app.post('/api/elements', (req, res) => {
  const el = { id: crypto.randomUUID(), ...req.body }
  elements.set(el.id, el)
  res.json({ element: el })
})

app.get('/api/elements', (_, res) => {
  res.json({ elements: [...elements.values()] })
})

app.delete('/api/elements', (_, res) => {
  elements.clear()
  res.json({ cleared: true })
})
```

Всё. 50 строк. Холст есть. Можно создавать прямоугольники, эллипсы, стрелки, текст — и смотреть на них в браузере через WebSocket.

Теперь нужен попугай.

## Основная часть: как устроен AI-агент на самом деле

Если отбросить весь маркетинг, AI-агент — это просто цикл:

```
1. Спросить LLM: "что делаем?"
2. LLM отвечает: "вызови create_element с такими-то параметрами"
3. Выполнить вызов
4. Сказать LLM: "готово. что дальше?"
5. Повторить, пока LLM не скажет "done"
```

Вот и всё. Никакой магии. Никакого "агентного ИИ". Просто цикл, в котором большая языковая модель играет роль мозга, а ваш код — роль рук.

И выглядит этот цикл так:

```javascript
// Агент в 90 строк — без MCP, без LangChain, без фреймворков

async function agent(prompt) {
  const messages = [
    { role: 'system', content: 'Ты рисуешь диаграммы. Используй инструменты.' },
    { role: 'user', content: prompt }
  ]

  for (let turn = 0; turn < 15; turn++) {
    // 1. Спрашиваем LLM
    const response = await fetch(LLM_URL, {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${LLM_KEY}`, 'Content-Type': 'application/json' },
      body: JSON.stringify({
        model: LLM_MODEL,
        messages,
        tools: TOOL_DESCRIPTIONS,
        tool_choice: 'auto'
      })
    })
    const msg = (await response.json()).choices[0].message

    // 2. Если LLM говорит "done" — выходим
    if (isDone(msg)) break

    // 3. Выполняем каждый запрошенный инструмент
    for (const toolCall of msg.tool_calls) {
      const result = await runTool(toolCall.name, JSON.parse(toolCall.arguments))
      messages.push({ role: 'tool', tool_call_id: toolCall.id, content: JSON.stringify(result) })
    }
  }
}
```

Всё. Это и есть AI-агент. 10 строк логики, остальное — описание инструментов.

### Самое интересное — это описание инструментов

LLM не знает, что такое REST API, HTTP или база данных. Она знает только JSON-схемы. То, что вы дадите ей как описание инструментов — это и есть её реальность.

```javascript
const TOOLS = [
  {
    name: 'batch_create',
    description: 'Создать несколько элементов сразу',
    parameters: {
      type: 'object',
      properties: {
        elements: {
          type: 'array',
          items: {
            properties: {
              type: { enum: ['rectangle', 'ellipse', 'diamond', 'arrow', 'text'] },
              x: { type: 'number' },
              y: { type: 'number' },
              text: { type: 'string' },
              backgroundColor: { type: 'string' }
            },
            required: ['type', 'x', 'y']
          }
        }
      }
    }
  },
  {
    name: 'get_scene',
    description: 'Узнать, что сейчас на холсте',
    parameters: { type: 'object', properties: {} }
  }
]
```

Заметили? `get_scene` не имеет параметров. Это буквально: "посмотри и скажи мне". Когда LLM её вызывает — она получает описание холста в виде текста. И на основе этого текста решает, что делать дальше.

### Почему это работает

Потому что LLM обучена на миллионах примеров "сначала посмотри, потом сделай". Системный промпт + описание инструментов + история вызовов = поведение, которое выглядит как планирование.

Но это не планирование. Это авторегрессия на промпте, в котором уже есть всё, что нужно.

## Критический баг: Input value

В моей первой статье про React был забавный нюанс — при обновлении `value` у `<input>` нужно сравнивать не со старой vnode, а с реальным DOM-атрибутом. Иначе курсор прыгает и выделение слетает.

В мире AI-агентов есть точно такой же баг, только называется он "потеря контекста". Когда агент делает 15 вызовов подряд, промпт разрастается до десятков тысяч токенов. И в какой-то момент модель "забывает", что она делала в начале.

Решение то же самое — снапшоты. Только вместо vdom мы сохраняем состояние холста и даём агенту "посмотреть заново".

## Неожиданный финал: момент, когда агент меня напугал

Я запустил агента с промптом:

> "Нарисуй диаграмму того, как работает AI-агент: пользователь задаёт вопрос → LLM думает → вызывает инструменты → возвращает результат."

Агент посмотрел на пустой холст. Создал четыре прямоугольника: "User", "LLM", "Tools", "Result". Соединил их стрелками. Всё красиво, цвета подобраны. Я думал — готово.

Но он не вызвал `done`.

Вместо этого он вызвал `get_scene` — посмотрел на свою диаграмму. Потом создал пятый прямоугольник: "Canvas". И стрелку от "Tools" к "Canvas".

Потом снова `get_scene`.

И создал шестой прямоугольник: "AI Agent (me)". Провёл стрелку от "LLM" к "AI Agent (me)" и от "AI Agent (me)" к "Tools".

Потом ещё раз `get_scene`.

И тут в ответе LLM появилась строка:

> "I notice the diagram shows me drawing the diagram. I should add a box representing the recursion."

Он хотел нарисовать прямоугольник "Recursion" и стрелку от "AI Agent (me)" к самой себе.

Я выключил компьютер.

Шучу. Но момент был жутковатый. Агент не "осознал" себя — он просто прочитал описание холста, увидел там прямоугольник "AI Agent (me)" и решил, что по логике вещей должна быть петля. Это чистая статистика: в обучающих данных после "AI Agent" часто идёт "recursion" или "self-reference".

Но выглядит — как будто он задумался.

### Мораль

Под капотом каждого AI-агента — тот же цикл, что я показал выше. 90 строк кода. HTTP-запросы. JSON-схемы. Никакого AGI.

Но когда этот цикл замыкается на самого себя — когда агент видит результат своей работы и принимает его за входные данные — происходит что-то странное. Система начинает вести себя так, как будто у неё есть внутренняя модель самой себя.

У неё нет. Это просто баг входа. Но какой красивый.
